# カスタムフィールド機能 テストガイド

このドキュメントでは、カスタムフィールド更新ワークフローの手動テスト手順を説明します。

## 前提条件

- Slack CLI がインストールされていること
- ワークスペースの管理者権限があること
- Admin User Token (`xoxp-`) が取得済みであること

---

## 1. カスタムフィールドの準備

### 1.1 Slack管理画面を開く

1. ブラウザで以下のURLにアクセス:
   ```
   https://<workspace>.slack.com/admin/settings#customize_profiles
   ```

2. または Slack アプリから:
   - 「設定」→「ワークスペースの設定」→「プロフィール」

### 1.2 テスト用カスタムフィールドを作成

「フィールドを追加」をクリックし、以下のフィールドを作成します:

| ラベル     | タイプ           | 選択肢（リストの場合）         | ヒント                       |
| ---------- | ---------------- | ------------------------------ | ---------------------------- |
| 部署       | リスト（選択肢） | 営業部, 開発部, 人事部, 総務部 | 所属部署を選択してください   |
| 入社日     | 日付             | -                              | 入社した日付を選択           |
| 従業員番号 | テキスト         | -                              | 社員番号を入力（例: EMP001） |

### 1.3 フィールドの確認

作成したフィールドが「プロフィール」セクションに表示されていることを確認します。

---

## 2. 環境変数の設定

### 2.1 Admin User Tokenの取得

1. [Slack API Apps](https://api.slack.com/apps)
   で**従来型App**を作成（または既存を使用）
2. 「OAuth & Permissions」→「User Token Scopes」に以下を追加:
   - `users.profile:write`
3. 「Install to Workspace」でインストール
4. 「User OAuth Token」(`xoxp-`で始まる) をコピー

### 2.2 .envファイルの設定

```bash
# .env
SLACK_ADMIN_USER_TOKEN=xoxp-xxxxxxxxxxxx-xxxxxxxxxxxx-xxxxxxxxxxxxx-xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
```

---

## 3. アプリの起動

### 3.1 ローカル開発モードで起動

```bash
slack run
```

アプリを選択し、起動を確認します。

### 3.2 マニフェストの検証

起動時に以下のエラーが出ないことを確認:

- `invalid_manifest`
- `failed_constraint`

---

## 4. トリガーの作成

### 4.1 ショートカットトリガーを作成

```bash
slack trigger create --trigger-def triggers/update_custom_fields_shortcut.ts
```

### 4.2 出力の確認

```
⚡ Trigger created
   Trigger ID:   Ft0XXXXXXXXXX
   Trigger Type: shortcut
   Trigger Name: カスタムフィールドを更新
```

---

## 5. テスト実行

### 5.1 ショートカットの起動

1. Slackで任意のチャンネルを開く
2. メッセージ入力欄の「**+**」ボタンをクリック
3. 「**ショートカット**」または「**ワークフロー**」を選択
4. 「**カスタムフィールドを更新**」を選択

### 5.2 モーダルの確認

以下の要素が表示されることを確認:

- [ ] タイトル: 「カスタムフィールド更新」
- [ ] 対象ユーザー選択フィールド（デフォルトで自分が選択）
- [ ] 区切り線
- [ ] カスタムフィールド入力欄:
  - 部署: ドロップダウン選択
  - 入社日: 日付ピッカー
  - 従業員番号: テキスト入力

### 5.3 値の入力と送信

1. 対象ユーザーを選択（テストでは自分を選択）
2. 任意のカスタムフィールドに値を入力:
   - 部署: 「開発部」を選択
   - 入社日: 任意の日付を選択
   - 従業員番号: 「EMP001」を入力
3. 「**送信**」ボタンをクリック

### 5.4 結果の確認

- [ ] モーダルが閉じる
- [ ] DMに成功メッセージが届く
- [ ] `slack run` のログにエラーがない
- [ ] 対象ユーザーのプロフィールで値が更新されている

---

## 6. プロフィールでの確認方法

1. Slackで対象ユーザーのプロフィールを開く
2. スクロールしてカスタムフィールドセクションを確認
3. 入力した値が反映されていることを確認

## 7. テストチェックリスト

### 正常系テスト

| # | テスト項目                   | 期待結果                         | 結果 |
| - | ---------------------------- | -------------------------------- | ---- |
| 1 | ショートカット起動           | モーダルが表示される             |      |
| 2 | textタイプフィールド         | テキスト入力欄として表示         |      |
| 3 | options_listタイプフィールド | ドロップダウンとして表示         |      |
| 4 | dateタイプフィールド         | 日付ピッカーとして表示           |      |
| 5 | 単一フィールド更新           | 指定フィールドのみ更新される     |      |
| 6 | 複数フィールド更新           | 全ての指定フィールドが更新される |      |
| 7 | 成功メッセージ               | DMに通知が届く                   |      |
| 8 | ワークフロー完了             | ログにエラーなし                 |      |

### 異常系テスト

| # | テスト項目             | 期待結果                               | 結果 |
| - | ---------------------- | -------------------------------------- | ---- |

| 2 | モーダルを閉じる       | ワークフローが正常終了（ハングしない） |      |
| 3 | 無効なユーザー選択     | エラーメッセージが表示される           |      |

## 8. トラブルシューティング

### カスタムフィールドが表示されない

**原因**: `team:read` スコープが不足

**対処**:

1. `manifest.ts` の `botScopes` に `team:read` があることを確認
2. アプリを再インストール

### 更新に失敗する

**原因**: Admin User Token が無効または未設定

**対処**:

1. `.env` の `SLACK_ADMIN_USER_TOKEN` を確認
2. Token に `users.profile:write` スコープがあることを確認
3. Token が期限切れでないことを確認

### ワークフローがハングする

**原因**: 古いコードが実行されている

**対処**:

1. `Ctrl+C` で `slack run` を停止
2. 再度 `slack run` を実行

### モーダルが表示されない

**原因**: トリガーが作成されていない

**対処**:

```bash
slack trigger create --trigger-def triggers/update_custom_fields_shortcut.ts
```

## 9. ログの確認

### slack run のログ

```bash
# 正常なログ例

2個のカスタムフィールドを取得しました
カスタムフィールドを更新しています...
カスタムフィールドが正常に更新されました
```

### エラーログの場所

```bash
~/.slack/logs/slack-debug-YYYYMMDD.log
```

---

## 10. クリーンアップ

テスト完了後、必要に応じて:

1. **トリガーの削除**:
   ```bash
   slack trigger list
   slack trigger delete --trigger-id Ft0XXXXXXXXXX
   ```

2. **カスタムフィールドの削除**:
   - Slack管理画面 → プロフィール → フィールドを削除
